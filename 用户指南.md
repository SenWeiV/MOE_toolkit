# MOESkills 用户指南

本文档面向实际使用 `MOESkills` 的终端用户，帮助你完成安装、配置、测试和排错。

## 1. 你需要准备什么

- 一台 macOS 电脑
- 本机已安装 `python3`
- 一枚由管理员发放的 API Key
- 云端地址

如果你是 beta 用户，通常会从管理员那里拿到两项信息：

- `server_url`
- `api_key`

例如：

```text
server_url = http://120.48.83.123:8080
api_key = sk_beta_xxx
```

## 2. 安装 MOESkills

推荐直接使用一键安装脚本：

```bash
curl -fsSL <SERVER_URL>/install.sh | \
  bash -s -- \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

安装完成后，默认会生成：

- `~/.local/bin/moeskills`
- `~/.local/bin/moe-connector`  
  说明：这是兼容旧版本的别名，不建议新用户继续使用
- `~/.moeskills/config.toml`
- `~/MOE Outputs`

如果你的 shell 里还找不到命令，先把下面这一行加到 shell 配置里：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

## 3. 配置与检查

如果你已经通过安装脚本传了 `--server-url` 和 `--api-key`，通常不需要再手动配置。

手动配置命令：

```bash
moeskills config set \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

查看当前配置：

```bash
moeskills config show --json
```

检查连通性：

```bash
moeskills doctor
```

正常情况下你应该看到：

- 云端服务可达
- `Authenticated: True`
- 输出目录正常

## 4. 最推荐的使用方式

对于大多数 agent、skill、自动化脚本，直接调用 CLI 就够了。

最常用的命令是：

```bash
moeskills run \
  --task "分析这个 CSV 并生成趋势图" \
  --attach ./sales.csv \
  --wait \
  --json
```

说明：

- `--task` 是自然语言任务描述
- `--attach` 可以重复传多个文件
- `--wait` 表示等待任务完成再返回
- `--json` 适合给 agent 或脚本消费

产物默认会下载到：

```text
~/MOE Outputs
```

## 5. 建议你先做这 3 个测试

### 5.1 健康检查

```bash
moeskills doctor
```

预期：

- 命令退出码为 `0`
- 显示云端可达
- 显示鉴权成功

### 5.2 CSV 图表测试

准备一个简单 CSV，例如 `sales.csv`：

```csv
month,value
1,10
2,20
3,30
```

执行：

```bash
moeskills run \
  --task "分析这个 CSV 并生成趋势图" \
  --attach ./sales.csv \
  --wait \
  --json
```

预期：

- 返回 `status = success`
- 返回 `run_id`
- `downloaded_paths` 中至少有：
  - `...summary.json`
  - `...chart.svg`

### 5.3 不支持任务测试

执行：

```bash
moeskills run \
  --task "帮我联网搜索最新 AI 新闻" \
  --wait \
  --json
```

预期：

- 返回结构化失败结果
- 退出码为 `4`
- `error_code` 或 `selection_reason` 体现 `unsupported_task` / `no_match`
- 不应误路由到 CSV/Excel 工具

## 6. 如果你要给 agent 直接调用

最稳的模式是让 agent 只用下面这几条命令：

```bash
moeskills config show --json
moeskills doctor --json
moeskills run --task "..." --attach ./file.csv --wait --json
moeskills runs get <RUN_ID> --json
moeskills artifacts list <RUN_ID> --json
moeskills artifacts download <RUN_ID> --json
moeskills registry search --json
```

这样做的好处是：

- 输出稳定
- 适合脚本和 agent 解析
- 不依赖宿主集成

## 7. 可选：接入 Codex CLI / Claude Code / OpenClaw

如果你不是直接让 agent 调 CLI，而是要把 `MOESkills` 挂进宿主，再用下面的命令。

### 7.1 Codex CLI

```bash
moeskills host install codex-cli
moeskills host doctor codex-cli
```

### 7.2 Claude Code

```bash
moeskills host install claude-code
moeskills host doctor claude-code
```

### 7.3 OpenClaw

```bash
moeskills host install openclaw
moeskills host doctor openclaw --workspace-path <OPENCLAW_WORKSPACE>
```

如果 OpenClaw 自动发现 workspace 失败，请显式传路径：

```bash
moeskills host install openclaw --workspace-path ~/.openclaw/workspace-your-agent
```

OpenClaw 产物会下载到：

```text
<OPENCLAW_WORKSPACE>/MOE Outputs
```

## 8. 常见命令速查

设置配置：

```bash
moeskills config set --server-url <SERVER_URL> --api-key <YOUR_API_KEY>
```

查看配置：

```bash
moeskills config show --json
```

检查服务：

```bash
moeskills doctor
```

执行任务：

```bash
moeskills run --task "分析这个 CSV 并生成趋势图" --attach ./sales.csv --wait --json
```

查看某次运行：

```bash
moeskills runs get <RUN_ID> --json
```

列出产物：

```bash
moeskills artifacts list <RUN_ID> --json
```

下载产物：

```bash
moeskills artifacts download <RUN_ID> --json
```

查工具注册表：

```bash
moeskills registry search --json
```

## 9. 常见问题

### 9.1 `moeskills: command not found`

通常是 `~/.local/bin` 不在 `PATH` 里。

先执行：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

然后重开一个终端。

### 9.2 `doctor` 失败

优先检查：

- `server_url` 是否正确
- API Key 是否完整
- 本机网络是否可访问云端地址

你可以先看当前配置：

```bash
moeskills config show --json
```

### 9.3 任务返回鉴权失败

重新写一次配置：

```bash
moeskills config set \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

然后再执行：

```bash
moeskills doctor
```

### 9.4 OpenClaw 里看不到效果

先检查：

- 目标 agent 是否至少启动过一次
- workspace 路径是否正确
- 是否执行过：

```bash
moeskills host doctor openclaw --workspace-path <OPENCLAW_WORKSPACE>
```

### 9.5 如何卸载

如果只是卸载宿主注册：

```bash
moeskills host uninstall codex-cli
moeskills host uninstall claude-code
moeskills host uninstall openclaw --workspace-path <OPENCLAW_WORKSPACE>
```

如果你是通过安装包安装的，也可以直接运行发布包中的 `uninstall.sh`。

## 10. 建议用户回传哪些测试信息

如果你在帮别人做 beta 测试，建议让他们回传：

- 使用的宿主类型
- `moeskills doctor` 输出
- 测试命令
- 返回的 `run_id`
- 生成的文件名
- 失败时的完整终端报错

这样排查会快很多。
