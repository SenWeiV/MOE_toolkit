# MOESkills 用户指南

本文档面向实际使用 `MOESkills` 的终端用户。  
目标是让你快速完成：

- 安装 CLI
- 配置 `server_url` 和 `api_key`
- 运行一次真实任务
- 用标准 CLI 命令完成检查、调试和测试

说明：

- 本指南已按当前仓库中的 `moeskills` CLI 更新，包含新增的交互 REPL 与 `runs wait` 命令。
- 文中命令均以当前已经实现的 `moeskills` 为准。

## 1. 你需要准备什么

- 一台 macOS 电脑
- 本机已安装 `python3`
- 一枚由管理员发放的 API Key
- 云端地址

你通常会从管理员那里拿到两项信息：

```text
server_url = http://120.48.83.123:8080
api_key = sk_beta_xxx
```

## 2. 安装方式

### 2.1 源码安装（适合本地开发与验证）

如果你就在这个仓库里直接使用 CLI，最稳的方式是从源码安装：

```bash
cd /Users/weisen/Documents/small-project/MOE_toolkit
python3 -m venv .venv
./.venv/bin/pip install -e .
./.venv/bin/moeskills --version
```

如果你已经有可用虚拟环境，也可以只执行：

```bash
python3 -m pip install -e .
moeskills --version
```

### 2.2 推荐：一键安装

```bash
curl -fsSL <SERVER_URL>/install.sh | \
  bash -s -- \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

这条命令会完成：

- 安装 `~/.local/bin/moeskills`
- 安装兼容别名 `~/.local/bin/moe-connector`
- 写入 `~/.moeskills/config.toml`
- 创建默认产物目录 `~/MOE Outputs`

### 2.3 发布包安装

如果你拿到的是发布包，也可以这样安装：

```bash
tar -xzf moeskills-macos.tar.gz
cd moeskills-release
bash install.sh \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

### 2.4 PATH 配置

如果安装完成后执行 `moeskills` 提示找不到命令，先把下面这行加入 shell 配置：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

然后重开终端。

## 3. 配置命令

如果你已经在安装脚本里传了 `--server-url` 和 `--api-key`，通常不需要再手动配置。

### 3.1 写入配置

```bash
moeskills config set \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

### 3.2 查看配置

```bash
moeskills config show --json
```

### 3.3 版本检查

```bash
moeskills --version
```

## 4. CLI 命令总览

当前对用户最重要的命令分为 7 组。

### 4.1 进入交互 REPL

如果你希望像命令台一样连续操作，可以直接执行：

```bash
moeskills
```

也可以显式进入交互模式，并预设本次会话的默认参数：

```bash
moeskills shell \
  --config-path ~/.moeskills/config.toml \
  --output-dir ./downloads
```

进入后常用内置命令有：

- `use config <PATH>`：记住默认 `--config-path`
- `use output-dir <PATH>`：记住默认 `--output-dir`
- `use workspace <PATH>`：记住默认 `--workspace-path`
- `use host-client <NAME>`：记住默认 `--host-client`
- `state`：查看当前会话默认值
- `exit` / `quit`：退出交互模式

### 4.2 健康检查

```bash
moeskills doctor
moeskills doctor --json
```

### 4.3 执行任务

```bash
moeskills run --task "..." --attach ./file.csv --wait --json
```

常用参数：

- `--task`：自然语言任务描述
- `--attach`：可重复传多个附件
- `--wait`：等待任务执行完成
- `--json`：输出机器可读 JSON，适合 agent 和脚本

### 4.4 查询或等待运行状态

```bash
moeskills runs get <RUN_ID> --json
moeskills runs wait <RUN_ID> --json
```

典型异步流程：

```bash
moeskills run --task "分析这个 CSV 并生成趋势图" --attach ./sales.csv --json
moeskills runs wait <RUN_ID> --json
```

`runs wait` 会轮询直到任务结束；如果任务成功，还会自动下载产物。

### 4.5 查询与下载产物

```bash
moeskills artifacts list <RUN_ID> --json
moeskills artifacts download <RUN_ID> --json
```

如果你想下载到自定义目录：

```bash
moeskills artifacts download <RUN_ID> --output-dir ./downloads --json
```

### 4.6 查询工具注册表

```bash
moeskills registry search --json
moeskills registry search --capability data_analysis --json
moeskills registry get pandas --json
moeskills registry manifest pandas --version 0.1.0 --json
```

### 4.7 直接调用云端 API

如果你希望保留 CLI 鉴权和配置，但手工调用底层 HTTP API：

```bash
moeskills api GET /v1/service/health --json
moeskills api GET /v1/registry/tools/search --json
```

带 JSON 请求体的例子：

```bash
moeskills api POST /v1/tasks/execute \
  --data @request.json \
  --json
```

其中 `request.json` 例如：

```json
{
  "task": "分析这个 CSV 并生成趋势图",
  "attachments": ["upload-id"],
  "session_id": "demo-session",
  "output_preferences": {}
}
```

## 5. 最推荐的使用方式

对于大多数 agent、skill、自动化脚本，直接用 CLI 就够了。

标准 one-shot 命令：

```bash
moeskills run \
  --task "分析这个 CSV 并生成趋势图" \
  --attach ./sales.csv \
  --wait \
  --json
```

这条命令最适合 agent 的原因：

- 不需要自己管理上传/轮询/下载多步流程
- 输出稳定
- 支持脚本解析
- 与宿主无关

产物默认会下载到：

```text
~/MOE Outputs
```

如果你不想在第一次调用里阻塞等待，也可以拆成两步：

```bash
moeskills run \
  --task "分析这个 CSV 并生成趋势图" \
  --attach ./sales.csv \
  --json

moeskills runs wait <RUN_ID> --json
```

## 6. 推荐测试流程

建议新用户按下面顺序测试。

### 6.1 测试 1：健康检查

```bash
moeskills doctor --json
```

预期：

- 退出码为 `0`
- `status = ok`
- `authenticated = true`

### 6.2 测试 2：CSV 图表

先准备一个简单 CSV，例如 `sales.csv`：

```csv
month,value
1,10
2,20
3,30
```

执行：

```bash
moeskills run \
  --task "分析这个 CSV 并生成趋势图" \
  --attach ./sales.csv \
  --wait \
  --json
```

预期：

- 返回 `status = success`
- 返回 `run_id`
- `downloaded_paths` 中至少有：
  - `...summary.json`
  - `...chart.svg`

### 6.3 测试 3：Excel 报表

如果你手头有 `sales.xlsx`，执行：

```bash
moeskills run \
  --task "读取这个 Excel 并生成 Excel 报表" \
  --attach ./sales.xlsx \
  --wait \
  --json
```

预期：

- 返回 `status = success`
- `downloaded_paths` 中包含：
  - `...summary.json`
  - `...report.xlsx`

### 6.4 测试 4：不支持任务

执行：

```bash
moeskills run \
  --task "帮我联网搜索最新 AI 新闻" \
  --wait \
  --json
```

预期：

- 返回结构化失败结果
- 退出码为 `4`
- 结果里出现 `unsupported_task` 或 `no_match`
- 不应误路由到 CSV/Excel 工具

## 7. 给 agent 的推荐调用范式

如果你要让另一个 agent 调用 `MOESkills`，建议只暴露下面这组稳定命令：

```bash
moeskills config show --json
moeskills doctor --json
moeskills run --task "..." --attach ./file.csv --wait --json
moeskills runs get <RUN_ID> --json
moeskills runs wait <RUN_ID> --json
moeskills artifacts list <RUN_ID> --json
moeskills artifacts download <RUN_ID> --json
moeskills registry search --json
moeskills api GET /v1/service/health --json
```

这样做的优势：

- stdout 结构稳定
- 适合非交互调用
- 便于串联进 shell、workflow、其他 agent

## 8. 可选：接入宿主

如果你不是直接让 agent 调 CLI，而是想把 `MOESkills` 挂到宿主中，可以使用 `host` 子命令。

### 8.1 Codex CLI

```bash
moeskills host install codex-cli
moeskills host doctor codex-cli
```

### 8.2 Claude Code

```bash
moeskills host install claude-code
moeskills host doctor claude-code
```

### 8.3 OpenClaw

```bash
moeskills host install openclaw
moeskills host doctor openclaw --workspace-path <OPENCLAW_WORKSPACE>
```

如果 OpenClaw 自动发现 workspace 失败，请显式传路径：

```bash
moeskills host install openclaw --workspace-path ~/.openclaw/workspace-your-agent
```

OpenClaw 产物会下载到：

```text
<OPENCLAW_WORKSPACE>/MOE Outputs
```

## 10. 常见问题

### 10.1 `moeskills: command not found`

通常是 `~/.local/bin` 不在 `PATH` 中。

执行：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

然后重开终端。

### 10.2 `doctor` 失败

优先检查：

- `server_url` 是否正确
- API Key 是否完整
- 云端地址是否可访问

先看配置：

```bash
moeskills config show --json
```

### 10.3 鉴权失败

重新写入配置：

```bash
moeskills config set \
  --server-url <SERVER_URL> \
  --api-key <YOUR_API_KEY>
```

然后再执行：

```bash
moeskills doctor
```

### 10.4 任务失败但不知道原因

建议按这个顺序排查：

```bash
moeskills run --task "..." --attach ./file.csv --wait --json
moeskills runs get <RUN_ID> --json
moeskills runs wait <RUN_ID> --json
moeskills artifacts list <RUN_ID> --json
```

重点看：

- `status`
- `error_code`
- `detail`
- `route_plan.selection_reason`

### 10.5 OpenClaw 里看不到效果

先确认：

- 目标 agent 至少启动过一次
- workspace 路径正确
- 已执行：

```bash
moeskills host doctor openclaw --workspace-path <OPENCLAW_WORKSPACE>
```

### 10.6 如何卸载

如果只是卸载宿主注册：

```bash
moeskills host uninstall codex-cli
moeskills host uninstall claude-code
moeskills host uninstall openclaw --workspace-path <OPENCLAW_WORKSPACE>
```

如果你是通过安装包安装的，也可以直接运行发布包中的 `uninstall.sh`。

## 11. 建议测试用户回传哪些信息

如果你在组织 beta 测试，建议让用户回传：

- 使用的宿主类型
- `moeskills doctor` 输出
- 实际执行的测试命令
- 返回的 `run_id`
- 生成的文件名
- 失败时的完整终端报错

这样排查会快很多。
